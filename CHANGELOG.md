# 更新日志

## 2.6 - 2026-05-17

### 新增

- 历史页新增当日详情概览，可快速查看当天记录数、跑步里程、健身部位和训练组数。
- 历史记录卡片展示更完整细节：跑步记录显示里程与配速，健身记录显示训练部位、动作、组数、重量或自重。
- 跑步打卡和历史编辑新增日期步进、公里数加减、常用配速快捷选择等移动端输入控件。
- 健身打卡日期改为可步进选择，减少手动输入日期的操作成本。

### 优化

- 保存跑步或健身打卡后，会先显示明确的成功反馈，再返回上一页。
- 历史记录编辑和删除后，会在历史页显示短暂反馈提示。

### 文档

- 新增 `DEVELOPER_GUIDELINES.md`，明确每次迭代的改动范围、版本号、更新日志、APK 产物、验证和 GitHub 发布规范。
- README 的体验 APK 下载链接更新到 `v2.6` Release。

## 2.5 - 2026-05-16

### 优化

- 健身计划展示页的动作重量文案从“默认重量”调整为“重量”。
- 编辑计划动作时新增“自重”快捷选择；未填写重量或选择“自重”时按自重保存。
- 重量输入增加校验，避免无效文本被误保存为 0 kg。

## 2.1 - 2026-05-15

### 修复

- 新增健身计划自动快照，覆盖安装或系统恢复时会保留用户原本编辑过的健身计划。
- 启动时如果检测到数据库中的训练计划回落到默认计划，会优先用已备份的训练计划快照恢复。
- 手动 JSON 备份也会携带训练计划快照，避免只恢复打卡记录和自定义标语、漏掉训练计划。

## 2.0 - 2026-05-15

### 新增

- 健身计划支持“自定义”分化，不再只限制为三分化或四分化。
- 训练日支持自定义主训练部位，例如“背”“肩前束”“肱二头”或用户自己的命名。
- 小板块和计划动作的训练部位支持自定义输入，并保留背、胸、腿、手臂快捷选择。
- 健身打卡页会根据当前训练计划动态展示可选择的分化/训练日。
- 健身雷达图会读取当前训练计划和历史打卡，动态展示自定义部位的训练频率。

### 优化

- 训练计划编辑动作改为弹窗编辑，不需要再滑回页面顶部修改。
- 编辑动作、小板块、训练日的弹窗改为深色设计，解决亮黄色背景配白字看不清的问题。
- 历史记录编辑健身打卡时，训练部位改为可编辑文本，兼容自定义部位。

### 修复

- 修复健身打卡页只显示四分化选项，无法跟随三分化或自定义计划变化的问题。
- 移除健身打卡摘要里固定“四分化”的文案，让摘要跟随实际选择的训练部位。

### 验证

- 已通过 `.\gradlew.bat assembleDebug` 构建验证。
